## 第一步：在 GitHub 上创建仓库（2 分钟）

### 操作步骤

1. **访问 GitHub**
   - 打开浏览器，访问：https://github.com
   - 确保已登录

2. **创建新仓库**
   - 点击右上角的 **"+"** 按钮
   - 选择 **"New repository"**

3. **填写仓库信息**

   ```
   Repository name: SimpleAccounting
   Description: 简单记账应用
   Public/Private: 选择任意（推荐 Public）
   ```

4. **重要！不要勾选以下选项**
   - ❌ Add a README file
   - ❌ Add .gitignore
   - ❌ Choose a license

5. **创建仓库**
   - 点击绿色按钮 **"Create repository"**

6. **复制仓库地址**

   创建成功后，页面会显示类似这样的地址：
   ```
   https://github.com/你的用户名/SimpleAccounting.git
   ```

   **记下这个地址！** 稍后需要用到。

   示例：`https://github.com/zhangsan/SimpleAccounting.git`

## 第二步：上传代码到 GitHub（5 分钟）

### 方法 A：使用自动脚本（推荐）

#### 步骤 2.1：运行上传脚本

**方式1：双击运行**
```
打开文件夹：c:\Users\Admins\Documents\记账
找到文件：upload_to_github.py
双击运行
```

**方式2：命令行运行**
```
打开 cmd 或 PowerShell
输入：
cd c:\Users\Admins\Documents\记账
python upload_to_github.py
```

#### 步骤 2.2：按照提示输入

脚本会询问你以下信息：

```
[1/5] 检查环境
[OK] Git 版本: 2.x.x

[2/5] 初始化 Git 仓库
[OK] Git 仓库初始化成功

[3/5] 添加文件
[OK] 文件添加成功

[4/5] 创建提交
[OK] 提交成功

[5/5] 配置 GitHub
请输入你的 GitHub 用户名: [输入你的用户名]
请输入仓库名称（默认: SimpleAccounting）: [按回车使用默认值]
```

**示例：**
```
请输入你的 GitHub 用户名: zhangsan
请输入仓库名称（默认: SimpleAccounting）: [直接按回车]
```

#### 步骤 2.3：确认仓库已创建

脚本会显示：
```
仓库地址：https://github.com/zhangsan/SimpleAccounting.git

重要提示：
1. 请先在 GitHub 上创建仓库：SimpleAccounting
2. 创建仓库时，不要初始化 README、.gitignore、License
3. 创建空仓库即可

是否已在 GitHub 上创建仓库？(Y/N):
```

如果你已经完成了第一步，输入：`Y`

#### 步骤 2.4：推送代码

脚本会继续执行：
```
关联远程仓库：https://github.com/zhangsan/SimpleAccounting.git
[OK] 远程仓库关联成功

是否推送代码到 GitHub？(Y/N): Y

推送代码到 GitHub...
```

#### 步骤 2.5：身份验证

**首次推送需要身份验证：**

**情况1：要求输入用户名和密码**
```
Username: zhangsan
Password: [这里不要输入GitHub登录密码]
```

**你需要创建 Personal Access Token：**

1. 访问：https://github.com/settings/tokens
2. 点击 **"Generate new token"** → **"Generate new token (classic)"**
3. 勾选权限：**repo**（全部勾选）
4. 设置过期时间：选择 **"90 days"** 或 **"No expiration"**
5. 点击 **"Generate token"**
6. **复制显示的 token**（类似：`ghp_xxxxxxxxxxxxxxxxxxxx`）
7. **回到命令行，粘贴 token 作为密码**

**⚠️ 重要：Token 只显示一次，一定要复制保存！**

#### 步骤 2.6：推送成功

成功后会看到：
```
Enumerating objects: 150, done.
Counting objects: 100% (150/150), done.
...
To https://github.com/zhangsan/SimpleAccounting.git
 * [new branch]      main -> main

✅ 上传成功！
```

### 方法 B：手动命令行（如果脚本失败）

如果自动脚本有问题，可以手动执行：

```bash
# 1. 打开命令行（cmd 或 PowerShell）

# 2. 进入项目目录
cd c:\Users\Admins\Documents\记账

# 3. 初始化 Git 仓库
git init

# 4. 添加所有文件
git add .

# 5. 创建提交
git commit -m "Initial commit: Simple Accounting App"

# 6. 关联远程仓库（替换为你的仓库地址）
git remote add origin https://github.com/你的用户名/SimpleAccounting.git

# 7. 重命名分支为 main
git branch -M main

# 8. 推送代码（使用 Token 作为密码）
git push -u origin main
```


你会看到：

**情况1：正在构建（首次推送）**

### 步骤 3.3：查看构建详情

点击构建任务，你会看到详细的构建步骤：

## 第四步：下载 APK（1 分钟）

### 步骤 4.1：进入构建任务页面

1. 在仓库页面点击 **"Actions"** 标签
2. 点击最新的构建任务（绿色 ✅）
3. 滚动到页面最底部

### 步骤 4.2：找到 Artifacts

你会看到：

```
Artifacts

📦 app-debug           下载    2.5 MB    2 分钟前
📦 app-release         下载    1.8 MB    2 分钟前
```

### 步骤 4.3：下载 APK

- **app-debug**：Debug 版本（用于测试）
- **app-release**：Release 版本（未签名，体积小）

点击文件名，APK 就会下载到你的下载文件夹！

**下载位置：**
```
C:\Users\你的用户名\Downloads\app-debug.apk
```

## 第五步：安装到手机（1-2 分钟）

### 方法 1：直接传输（最简单）

#### 步骤 5.1：传输 APK 到手机

**方式A：通过 USB**
1. 用 USB 线连接手机和电脑
2. 在电脑上找到下载的 APK 文件
3. 复制到手机的"下载"文件夹

**方式B：通过微信/QQ**
1. 在电脑上打开微信/QQ电脑版
2. 发送 APK 文件到"文件传输助手"
3. 在手机上接收并保存

**方式C：通过网盘**
1. 上传 APK 到百度网盘/阿里云盘等
2. 在手机上下载

#### 步骤 5.2：在手机上安装

1. 在手机上找到 APK 文件（通常在"下载"或"文件管理"中）
2. 点击 APK 文件
3. 如果提示"允许安装未知应用"：
   - 选择"设置"
   - 找到"允许安装未知应用"
   - 选择"允许"
4. 点击"安装"

#### 步骤 5.3：打开应用

安装完成后：
- 点击"打开"
- 或在桌面找到"简单记账"图标
- 开始使用！

### 方法 2：使用 ADB（高级用户）

如果你已配置 ADB：

```bash
# 1. 启用手机 USB 调试（开发者选项）
# 2. 用 USB 连接手机

# 3. 安装 APK
adb install app-debug.apk

# 4. 打开应用
```

## ❓ 遇到问题？

### 问题1：推送时提示认证失败

**解决：**
- 不要使用 GitHub 登录密码
- 使用 Personal Access Token
- 创建方式：https://github.com/settings/tokens

### 问题2：构建失败

**检查：**
- 代码是否正确推送
- 查看 Actions 页面的错误日志
- 检查 `.github/workflows/build.yml` 是否存在

### 问题3：找不到 APK 下载链接

**解决：**
- 等待构建完成（所有步骤都是 ✅）
- 滚动到构建页面最底部
- 查看 "Artifacts" 部分
