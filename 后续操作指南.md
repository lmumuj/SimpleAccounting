
### 步骤 2：配置 GitHub 仓库

**在 GitHub 上创建仓库：**

1. 打开浏览器，访问：https://github.com/new
2. 填写信息：
   - **Repository name**: `SimpleAccounting`
   - **Description**: 简单记账应用
   - **Public/Private**: 选择 Public（推荐）
3. **重要：不要勾选**以下选项：
   - ❌ Add a README file
   - ❌ Add .gitignore
   - ❌ Choose a license
4. 点击 **"Create repository"**

创建成功后，你会看到仓库地址，类似：
```
https://github.com/你的用户名/SimpleAccounting.git
```

**记住这个地址！**

---

### 步骤 3：关联远程仓库

回到命令行，执行：

```bash
cd c:\Users\Admins\Documents\记账

# 关联远程仓库（替换为你的实际地址）
git remote add origin https://github.com/你的用户名/SimpleAccounting.git
```

例如，如果你的用户名是 `zhangsan`：
```bash
git remote add origin https://github.com/zhangsan/SimpleAccounting.git
```

---

### 步骤 4：推送到 GitHub

```bash
# 重命名分支为 main
git branch -M main

# 推送代码
git push -u origin main
```

**首次推送时需要身份验证：**

如果提示输入用户名和密码：
- **Username**: 你的 GitHub 用户名
- **Password**: **不是你的 GitHub 登录密码！**

**你需要创建 Personal Access Token：**

1. 访问：https://github.com/settings/tokens
2. 点击 **"Generate new token"** → **"Generate new token (classic)"**
3. 勾选权限：**repo**（全部勾选）
4. 设置过期时间：选择 **"90 days"** 或 **"No expiration"**
5. 点击 **"Generate token"**
6. **复制显示的 token**（类似：`ghp_xxxxxxxxxxxxxxxxxxxx`）
7. **回到命令行，粘贴 token 作为密码**

⚠️ **Token 只显示一次，一定要复制保存！**

## 📦 步骤 7：下载 APK

1. 点击最新的绿色构建任务
2. 滚动到页面底部
3. 在 **"Artifacts"** 部分，找到：
   - **app-debug**（测试版）
   - **app-release**（未签名）
4. 点击文件名下载 APK

## ❓ 常见问题

### Q1: Git commit 命令总是报错？

**解决方法：**

尝试以下命令之一：

```bash
# 方法1：使用空格分隔
git commit -m Initial

# 方法2：使用简单文本
git commit -m ""

# 方法3：直接回车（不写消息）
git commit
```

### Q2: 推送时提示认证失败？

**解决：**
- 使用 Personal Access Token，不要用 GitHub 登录密码
- Token 创建：https://github.com/settings/tokens
- 勾选 repo 权限

### Q3: 找不到 APK 下载链接？

**解决：**
- 确保所有构建步骤都是 ✅ 绿色对勾
- 滚动到构建页面最底部
- 查看 "Artifacts" 部分

### Q4: 构建失败怎么办？

**解决：**
- 查看 Actions 页面的错误日志
- 检查 `.github/workflows/build.yml` 是否存在
- 确认代码已正确推送
